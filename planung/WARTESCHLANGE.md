# WARTESCHLANGE

Ideen und spaetere Aufgaben. Noch nicht sofort umsetzen, erst nach Stabilisierung.

## Funktionen

- [ ] Rollen/Rechte fuer Admin-Ansicht und normale Ansicht.
- [ ] Import-Assistent fuer vorhandene Inventarlisten.
- [ ] Exportprofile fuer Notion, Excel und einfache Archiv-ZIPs.
- [ ] Dashboard mit echten Kennzahlen: offene Tickets, fehlende Pflichtsoftware, Garantieablauf, unvollstaendige Netzwerkdaten.
- [ ] Historie pro Asset: Aenderungen, Scans, Tickets, Notizen.

## Technik

- [ ] JavaScript in Module aufteilen: Core, Datenzugriff, UI, Wizard, Dashboard, Admin, Scanner-Integration.
- [ ] Automatisierte Smoke-Tests fuer Browser-Workflows einfuehren.
- [ ] CSV-Zugriff langfristig optional durch SQLite ersetzen.
- [ ] Lokale Konfiguration einfuehren, z. B. Port, Datenpfad, Backup-Aufbewahrung.
- [ ] Release-Prozess definieren: Version bump, Syntaxcheck, ZIP/Export, Changelog.

## Aufraeumen spaeter

- [ ] Alte Dokumentationsdateien archivieren, sobald `HISTORIE.md` existiert.
- [ ] Demo-Daten von echten Daten trennen.
- [ ] Stammdaten-Dateien vereinheitlichen und validieren.
- [ ] UI-Texte und Begriffe konsistent machen: Asset, Geraet, Hardware, Inventar.
